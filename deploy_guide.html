0 部署前检查清单

  • 代码已推送到 GitHub 公开仓库 sankenluo/EVAP
  • 入口文件 app.py 在仓库根目录
  • requirements.txt 已固定所有依赖版本
  • .streamlit/config.toml 已配置
  • 应用无 Secrets / 环境变量依赖（无需额外配置）
  • 本地冒烟测试通过（HTTP 200，无报错）
  • CoolProp 8.0.0 有 Linux x86_64 wheel（兼容 Python 3.12+）

1 部署配置参数

在 Streamlit Cloud 创建应用时，填写以下配置：

配置项 值
Repository sankenluo/EVAP
Branch main
Main file path app.py
App URL (custom) 可留空，或自定义短链接
Python version 3.12（推荐）或 3.13
Secrets 无需配置（应用不使用 Secrets）
Advanced settings 保持默认即可
为什么选 Python 3.12？
CoolProp 8.0.0 提供 cp312-abi3 格式的 Linux wheel，兼容 Python 3.12 及以上版本。本地测试在 Python 3.13 上通过，Streamlit Cloud 选 3.12 最为稳妥。

2 部署操作步骤

步骤 1：访问 Streamlit Cloud

https://share.streamlit.io
↓

步骤 2：使用 GitHub 账号登录

点击 "Sign in with GitHub"，授权 Streamlit 访问你的 GitHub 仓库

↓

步骤 3：创建新应用

点击右上角 "New app" → 选择 "From existing repo"

↓

步骤 4：填写配置

  • Repository: 选择 sankenluo/EVAP
  • Branch: main
  • Main file path: app.py
  • Python version: 3.12
↓

步骤 5：点击 Deploy!

等待 3-5 分钟，Streamlit Cloud 会自动安装依赖并启动应用

部署成功后：
应用将获得一个公开 URL，格式为：
https://[app-name].streamlit.app
任何人都可以通过此 URL 访问应用，无需登录。

3 注意事项

资源限制：Streamlit Community Cloud 免费版限制：
• 每个应用 1GB 内存
• 应用闲置 7 天后自动休眠（再次访问时自动唤醒，约 30 秒）
• 每月有有限的构建时长
更新应用：每次向 GitHub 仓库 main 分支推送新提交时，Streamlit Cloud 会自动检测并重新部署。也可在管理面板手动点击 "Reboot" 或 "Delete and redeploy"。
CoolProp 依赖：如果 CoolProp 安装失败（极低概率），应用仍可正常运行——refrigerant.py 内置了多项式后备方案，会在 CoolProp 不可用时自动切换。
config.toml 说明：当前 .streamlit/config.toml 中的 port 和 headless 设置在 Streamlit Cloud 上会被自动覆盖，不影响部署。
